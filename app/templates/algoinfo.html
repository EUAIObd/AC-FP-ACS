Algorithm Information

Click on an algorithm to learn more about its history, process, and use cases.

Type: Symmetric block cipher
Key Sizes: 128, 192, or 256 bits
Summary: AES is the most widely used symmetric encryption algorithm today. It is fast, secure, and standardized by NIST.
Use Cases: File encryption, secure messaging, VPNs, disk encryption.
Type: Symmetric block cipher
Key Size: 56 bits
Summary: DES was the first widely adopted symmetric encryption standard. It is now considered insecure due to its short key length.
Use Cases: Historical/legacy systems, educational purposes.
Type: Symmetric block cipher
Key Sizes: 112 or 168 bits
Summary: 3DES applies DES three times to each data block. It is more secure than DES but slower and now being phased out.
Use Cases: Legacy systems, payment industry (historically).
Type: Asymmetric (public-key) encryption
Key Sizes: 1024, 2048, 3072, 4096 bits (2048+ recommended)
Summary: RSA is the most widely used public-key algorithm for secure data transmission and digital signatures.
Use Cases: Secure email, SSL/TLS, digital signatures, key exchange.
Type: Asymmetric (public-key) encryption
Key Sizes: 160-521 bits (much shorter than RSA for same security)
Summary: ECC provides strong security with smaller keys, making it efficient for mobile and IoT.
Use Cases: Digital signatures, secure messaging, cryptocurrencies.
Type: Cryptographic hash function
Output: 256 bits
Summary: SHA-256 is widely used for data integrity, digital signatures, and blockchain.
Use Cases: Password hashing, file integrity, blockchain.
Type: Cryptographic hash function
Output: 512 bits
Summary: SHA-512 is similar to SHA-256 but produces a longer hash, offering higher security for some applications.
Use Cases: File integrity, digital signatures, high-security applications.
Type: Cryptographic hash function
Output: 128 bits
Summary: MD5 is fast but no longer considered secure against collisions. Use only for non-security-critical checks.
Use Cases: Checksums, legacy systems, file verification (non-secure).
Type: Cryptographic hash function
Output: Variable (up to 512 bits)
Summary: BLAKE2 is a modern, fast, and secure hash function, designed as an alternative to MD5 and SHA.
Use Cases: Password hashing, file integrity, digital signatures.
